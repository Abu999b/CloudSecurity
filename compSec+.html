üîê 1. Threats, Attacks & Vulnerabilities ‚ñ∂
Malware
Malicious software like viruses, worms, Trojans, ransomware. Antivirus and regular updates help prevent it.
Phishing
Social engineering via email to trick users into revealing credentials or installing malware.
DoS / DDoS
Denial of Service attacks overwhelm a system; DDoS uses multiple sources. Mitigate with firewalls/load balancers.
Zero-day
An unknown vulnerability with no patch. Detected via anomaly-based IDS or behavioral monitoring.
Brute-force attack
Tries all possible password combinations. Use account lockout and complex passwords to prevent.
Insider threat
A trusted person misusing access. Least privilege and auditing help reduce risk.
SQL Injection
Inserting malicious SQL in input fields. Prevent with input validation and parameterized queries.
Botnet
Network of infected devices controlled by an attacker. Often used in DDoS attacks.
üîí 2. Architecture & Design ‚ñ∂
Defense in Depth
Layered security (firewalls, antivirus, access controls, etc.) to reduce attack risk.
Zero Trust
Never trust, always verify‚Äîevery access request is authenticated.
DMZ (Demilitarized Zone)
Public-facing services (like web servers) placed between internal and external networks.
Segmentation
Splitting networks for better security (e.g., VLANs, subnets). Limits lateral movement.
Cloud Models
IaaS = infrastructure, PaaS = platform for devs, SaaS = full applications.
On-Prem vs Cloud
On-prem offers control; cloud offers scalability and offloading. Hybrid is a mix.
IoT Security
IoT devices must be patched and monitored due to weak defaults and exposure.
üßë‚Äçüíº 3. Implementation ‚ñ∂
Firewall
Filters traffic based on rules (IP, port, protocol). Can be hardware or software.
VPN
Encrypted tunnel for remote access. Uses protocols like IPsec or SSL.
TLS/SSL
Encrypts web traffic (HTTPS). TLS is newer and more secure than SSL.
MFA (Multi-Factor Auth)
Uses two or more types: something you know, have, or are (e.g., password + OTP).
NAC (Network Access Control)
Restricts devices based on security posture (e.g., antivirus, patches).
SIEM
Security Information and Event Management‚Äîcentralized log analysis and alerts.
IDS/IPS
IDS detects attacks; IPS prevents them. Can be network-based or host-based.
üß™ 4. Operations & Incident Response ‚ñ∂
Incident Response
Steps: Preparation, Identification, Containment, Eradication, Recovery, Lessons Learned.
Forensics
Investigating attacks using logs, memory, and disk images. Maintain chain of custody.
SOAR
Security Orchestration, Automation, and Response‚Äîautomates incident handling.
Threat Intelligence
Info about threat actors and tools. Can be strategic (long-term) or tactical (real-time IOCs).
Data Loss Prevention
DLP tools monitor sensitive data and prevent unauthorized transfers or leaks.
Backup Strategies
Follow 3-2-1 rule: 3 copies, 2 media types, 1 offsite. Test restores regularly.
üìú 5. Governance, Risk & Compliance ‚ñ∂
Risk Assessment
Identifies threats and vulnerabilities to calculate risk. Includes qualitative and quantitative methods.
Security Policy
Documents rules for behavior, access, and system usage. Enforced through training and audits.
Privacy vs Security
Privacy = protection of personal data; Security = protection of systems and data.
GDPR, HIPAA, PCI-DSS
Examples of regulations that enforce privacy and security controls by law.
Acceptable Use Policy
Defines what users are allowed to do on organizational systems. Prevents misuse.
Least Privilege
Users get only the access they need. Reduces damage if account is compromised.
Change Management
Ensures all changes are documented, approved, and tested. Reduces misconfigurations.
